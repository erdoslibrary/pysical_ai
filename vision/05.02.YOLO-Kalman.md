# YOLOv8 실시간 객체 탐지 — 사전 지식 & 심화 가이드

## 1. 이 코드를 짜기 전에 알아야 할 것들

### 1-1. 컴퓨터 비전 기초

| 개념 | 핵심 내용 |
|------|-----------|
| **픽셀 좌표계** | 이미지 좌상단이 `(0, 0)`. 오른쪽·아래로 갈수록 값이 커짐 |
| **BGR vs RGB** | OpenCV는 BGR 채널 순서를 사용. `matplotlib`으로 시각화할 때 `[:,:,::-1]`로 변환 필요 |
| **프레임 / FPS** | 1초에 몇 장의 이미지를 처리하는가. 로봇 실시간 제어에는 최소 15 FPS 이상 권장 |
| **Bounding Box** | 탐지된 물체를 감싸는 직사각형. `xyxy`(좌상단·우하단) 또는 `xywh`(중심·크기) 표현 |
| **Confidence Score** | 모델이 "이게 맞다"고 확신하는 정도 (0~1). 일반적으로 0.5 이상만 유효로 취급 |

### 1-2. 딥러닝 객체 탐지 구조 이해

```
이미지 입력
    │
    ▼
Backbone (특징 추출 — CSPDarknet, C2f 블록)
    │  이미지에서 엣지·질감·형태 등의 특징맵 추출
    ▼
Neck (FPN / PAN — 다중 스케일 통합)
    │  작은 객체~큰 객체 모두 잡기 위해 다양한 해상도 특징 결합
    ▼
Head (예측 — Anchor-free)
    │  각 그리드 셀마다 [x, y, w, h, conf, class...] 예측
    ▼
NMS (Non-Maximum Suppression)
    │  겹치는 박스 중 신뢰도가 가장 높은 것만 남김
    ▼
최종 탐지 결과
```

**1-Stage vs 2-Stage 차이**

| | 1-Stage (YOLO, SSD) | 2-Stage (Faster R-CNN) |
|--|--|--|
| 처리 방식 | 위치 + 분류를 동시에 예측 | 후보 영역 추출 → 분류 순차 처리 |
| 속도 | 빠름 (실시간 가능) | 느림 |
| 정확도 | 상대적으로 낮음 | 상대적으로 높음 |
| 로봇 적합성 | ✓ 적합 | 라즈베리파이 등 엣지 장치에서 실시간 불가 |

### 1-3. OpenCV 웹캠 파이프라인

```python
cap = cv2.VideoCapture(0)      # 카메라 연결 (0 = 내장 웹캠)
ret, frame = cap.read()        # 프레임 1장 읽기 (ret=성공 여부)
cv2.imshow("window", frame)    # 화면에 출력
cv2.waitKey(1)                 # 키 입력 대기 (1ms). 없으면 창이 멈춤
cap.release()                  # 카메라 해제
cv2.destroyAllWindows()        # 모든 창 닫기
```

### 1-4. YOLOv8 모델 크기별 선택 기준

| 모델 | 크기 | 속도 | 정확도 | 권장 환경 |
|------|------|------|--------|-----------|
| `yolov8n.pt` | 6 MB | 최고 | 낮음 | 라즈베리파이, 젯슨 나노 |
| `yolov8s.pt` | 22 MB | 빠름 | 보통 | 일반 로봇 |
| `yolov8m.pt` | 52 MB | 보통 | 좋음 | 고성능 PC |
| `yolov8l/x.pt` | 87~136 MB | 느림 | 최고 | 연구용 |

### 1-5. 주요 predict() 파라미터

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `conf` | 0.25 | 신뢰도 임계값. 높이면 거짓 양성 감소 |
| `iou` | 0.7 | NMS IOU 임계값. 낮추면 겹치는 박스 더 적극 제거 |
| `classes` | None | 탐지할 클래스 인덱스 리스트. `[0]`이면 사람만 탐지 |
| `imgsz` | 640 | 추론 해상도. `320`으로 낮추면 약 2배 빨라짐 |
| `stream` | False | `True`시 메모리 효율적인 제너레이터 반환 (실시간 권장) |
| `verbose` | True | False로 설정하면 터미널 로그 비활성화 |

# 화면에 사람이 n명 이상이면 경보

    print("경고: 사람 3명 이상 감지!")

### 3-2. ROI (Region of Interest) 필터링
전체 화면이 아닌 **관심 영역 안에서만** 탐지 — 불필요한 감지 차단.
```python
roi = frame[100:400, 200:600]   # y_start:y_end, x_start:x_end
results = model.predict(roi, ...)
```

### 3-3. Kalman Filter로 바운딩 박스 안정화
YOLO 탐지는 프레임마다 떨림이 있음. 칼만 필터로 `(cx, cy)`를 스무딩하면
부드러운 추적이 가능. (→ `05.02.YOLO-Kalman.ipynb` 참고)

### 3-5. 거리 추정 (단안 카메라)

distance = (실제_너비 × focal_length) / 픽셀_너비
