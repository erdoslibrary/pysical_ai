## 1. 왜 카메라 모델이 필요한가?

카메라는 **3D 세계를 2D 이미지**로 변환합니다.  
이 변환 과정에서 한 가지 중요한 정보가 사라집니다 → **깊이(Depth, Z축)**

```
3D 세계 (X, Y, Z) ──→ 카메라 ──→ 2D 이미지 (u, v 픽셀)
                                        ↑
                               깊이 정보 손실!
```

**로봇이 카메라 모델을 사용하는 이유:**
- 2D 픽셀 좌표 → 실제 3D 공간 위치를 역으로 계산하기 위해
- 물체까지의 거리, 방향, 크기를 수학적으로 정확히 파악하기 위해

> **비유**: 지도를 보면 건물 위치(x, y)는 알 수 있지만, 건물 높이(z)는 별도 정보가 필요한 것과 같습니다.

## 2. 핀홀 카메라 모델

### 2.1 기본 원리

**"빛은 직선으로 나아간다"**는 단순한 원리에서 출발합니다.

```
물체
 │
 │ 빛 (직선)
 │
[구멍 (Pinhole)]
 │
 │
[이미지 센서]  ← 여기에 상이 맺힘
```

| 구성 요소 | 설명 |
|-----------|------|
| 바늘구멍(Pinhole) | 빛이 통과하는 단 하나의 점 |
| 광학 중심 | 바늘구멍의 위치 = 카메라의 원점 |
| 이미지 평면(센서) | 빛이 도달해 상이 맺히는 곳 |

### 2.2 조리개 (Aperture) — 구멍 크기의 딜레마

| 구멍 크기 | 빛의 양 | 이미지 선명도 |
|-----------|---------|--------------|
| 크다 | 많음 | **흐릿** (여러 방향의 빛이 섞임) |
| 작다 | 적음 | **선명** (한 방향 빛만 통과) |

**핀홀 모델의 수학적 가정**: 구멍 크기 = **0** (이상적인 점)  
→ 하지만 현실에서는 구멍이 너무 작으면 이미지가 너무 어두워짐

### 2.3 렌즈 — 딜레마의 해결책

렌즈는 **큰 구멍 + 선명한 이미지** 두 마리 토끼를 잡습니다.

```
물체의 한 점에서 나온 여러 빛줄기
         ↓
    [렌즈] ← 빛을 굴절시켜 한 점으로 모아줌
         ↓
    이미지 센서의 한 점에 수렴
```

**새로운 문제**: 렌즈 굴절 특성 때문에 **왜곡(Distortion)** 발생  
→ 이를 수학적으로 측정·보정하는 것이 **캘리브레이션**

## 3. 초점 거리 (Focal Length)

### 3.1 개념

**초점 거리 $f$**: 광학 중심(구멍)과 이미지 센서 사이의 물리적 거리

```
광학 중심 ←──── f (mm) ────→ 이미지 센서
```

| 초점 거리 | 특성 | 용도 |
|-----------|------|------|
| 길다 (큰 f) | 좁은 범위, 물체 크게 보임 | 망원 렌즈 |
| 짧다 (작은 f) | 넓은 범위, 물체 작게 보임 | 광각 렌즈 |

**렌즈 공식** (물체 거리 $z$, 상 거리 $z'$):

$$\frac{1}{f} = \frac{1}{z'} - \frac{1}{z}$$

### 3.2 픽셀 단위 초점 거리 ($f_x$, $f_y$)

컴퓨터에서는 물리 단위(mm) 대신 **픽셀 단위**를 사용합니다.

$$f_x = \frac{f}{d_x}, \quad f_y = \frac{f}{d_y}$$

| 기호 | 단위 | 의미 |
|------|------|------|
| $f$ | mm | 물리적 초점 거리 |
| $d_x, d_y$ | mm/pixel | 픽셀 하나의 실제 크기 |
| $f_x, f_y$ | pixel | 코드에서 쓰는 초점 거리 |

**예시 계산**:
```python
F_MM = 4.0    # 렌즈 초점 거리 4mm
DX   = 0.004  # 픽셀 가로 크기 0.004mm/pixel

fx = F_MM / DX  # = 4.0 / 0.004 = 1000 pixel
# fx=1000 의 의미: 초점 거리가 픽셀 1000개 길이에 해당
```

**왜 $f_x$, $f_y$ 두 개?**  
픽셀이 정확히 정사각형이 아닐 수 있기 때문입니다. 대부분의 현대 카메라는 $f_x \approx f_y$

## 4. 주점 (Principal Point)

**주점 $(c_x, c_y)$**: 카메라 렌즈의 광축이 이미지 센서와 수직으로 만나는 픽셀 좌표

```
이미지 센서
┌─────────────────────┐
│                     │
│         ×           │  ← 이상적 주점 (이미지 중앙)
│     (cx, cy)        │
│                     │
└─────────────────────┘
```

| 상황 | 주점 위치 |
|------|-----------|
| 이상적인 카메라 | 이미지 정중앙 `(width/2, height/2)` |
| 실제 카메라 | 제조 오차로 인해 중앙에서 약간 벗어남 |

> **왜 중요한가?** 주점이 틀리면 로봇이 물체 위치를 잘못 파악합니다.  
> 640×480 해상도 카메라의 이상적 주점: `cx=320, cy=240`

## 5. 카메라 행렬 K (내부 파라미터)

### 5.1 K 행렬이란?

카메라의 고유한 광학적 특성을 담은 **3×3 행렬**입니다.  
카메라를 바꾸면 K도 달라집니다. 같은 카메라라면 K는 항상 동일합니다.

$$K = \begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}$$

```python
# 예시: 640×480 해상도, fx=fy=1000px
K = np.array([
    [1000.,    0.,  320.],
    [   0., 1000.,  240.],
    [   0.,    0.,    1.]
])
```

### 5.2 K를 사용한 3D → 2D 투영

**3D 카메라 좌표 → 2D 픽셀 좌표** 변환 공식:

$$u = f_x \cdot \frac{X}{Z} + c_x, \quad v = f_y \cdot \frac{Y}{Z} + c_y$$

```python
X, Y, Z = 0.3, 0.1, 2.0  # 3D 카메라 좌표 (단위: m)

# 방법 1: 공식 직접 사용
u = fx * (X / Z) + cx   # = 470.0 px
v = fy * (Y / Z) + cy   # = 290.0 px

# 방법 2: 행렬 연산
point_norm = np.array([X/Z, Y/Z, 1])
pixel = K @ point_norm   # [470, 290, 1]
```

### 5.3 역방향: 2D 픽셀 → 3D (역투영)

깊이 Z를 **알고 있을 때**만 역으로 계산 가능합니다.

```python
x_norm = (u - cx) / fx   # 정규화 이미지 좌표
y_norm = (v - cy) / fy

X_real = x_norm * Z      # 실제 3D 좌표
Y_real = y_norm * Z

# 또는 행렬로: K_inv @ pixel * Z
```

> **핵심**: 깊이 Z 없이는 정확한 역투영 불가 → 깊이 카메라나 다른 방법 필요

## 6. 좌표계 4단계 변환

로봇 비전에서 "픽셀 하나가 실제 공간 어디에 있는가"를 알기까지 4단계를 거칩니다.

```
① 월드 좌표계     ②카메라 좌표계    ③이미지 좌표계    ④픽셀 좌표계
  (World Frame)  →(Camera Frame)→  (Image Plane) →  (Pixel Frame)
  방의 구석 기준    카메라 렌즈 기준   투영된 2D 평면     코드의 (u,v)
```

| 단계 | 사용 변환 | 관련 파라미터 |
|------|----------|--------------|
| ① → ② | 회전 + 이동 | **외부 파라미터** `[R|t]` |
| ② → ③ | 핀홀 투영 | **내부 파라미터** `K` |
| ③ → ④ | 픽셀 스케일링 | `fx, fy, cx, cy` |

**역방향 (픽셀 → 월드 좌표):**
$$\text{픽셀}(u,v) \xrightarrow{K^{-1}} \text{정규화 좌표} \xrightarrow{\times Z} \text{카메라 좌표} \xrightarrow{[R|t]^{-1}} \text{월드 좌표}$$

### 중요: 로봇 vs 카메라 좌표축 차이

```
로봇 본체 좌표계:     카메라 광학 좌표계:
  X = 앞 방향           Z = 앞 방향 (광축)
  Y = 왼쪽              X = 오른쪽
  Z = 위                Y = 아래 (이미지 y축과 동일)
```

> 두 좌표계가 90도 다릅니다! 혼용하면 위치 계산 오류 발생.

### 깊이 추정 — 닮음 삼각형 원리

크기를 아는 물체가 있으면 카메라 거리를 계산할 수 있습니다.

$$Z = \frac{X_{real} \times f}{x_{pixel} \times d_x}$$

```python
X_real_mm = 100.0   # 물체 실제 크기 100mm
x_pixel   = 50.0    # 이미지에서 50픽셀 차지
F_MM      = 4.0     # 초점 거리 4mm
DX        = 0.004   # 픽셀 크기 0.004mm/px

Z = X_real_mm * F_MM / (x_pixel * DX)  # = 8000mm = 8m
```

## 7. 렌즈 왜곡의 종류

### 7.1 방사 왜곡 (Radial Distortion)

렌즈 모양 때문에 이미지 **가장자리로 갈수록 직선이 곡선**으로 변하는 현상

```
배럴 왜곡 (Barrel)        핀쿠션 왜곡 (Pincushion)
┌───────────────┐         ┌───────────────┐
│  ╭─────────╮  │         │  ╱─────────╲  │
│  │         │  │         │ ╱           ╲ │
│  │         │  │   vs    │ │           │ │
│  ╰─────────╯  │         │ ╲           ╱ │
└───────────────┘         │  ╲─────────╱  │
                          └───────────────┘
    광각 렌즈에서 발생           망원 렌즈에서 발생
    이미지가 볼록하게 팽창       이미지가 오목하게 수축
```

수학적 모델:
$$x_{dist} = x(1 + k_1 r^2 + k_2 r^4 + k_3 r^6)$$

여기서 $r$ = 주점에서 해당 픽셀까지의 거리

### 7.2 접선 왜곡 (Tangential Distortion)

렌즈와 이미지 센서가 **완벽하게 수평을 이루지 못해** 발생  
이미지가 비스듬하게 기울어져 보임

```python
# 왜곡 계수 벡터: dist = [k1, k2, p1, p2, k3]
#   k1, k2, k3 → 방사 왜곡 계수
#   p1, p2     → 접선 왜곡 계수
dist = np.array([k1, k2, p1, p2, k3])
```

| 왜곡 계수 | 종류 | 설명 |
|-----------|------|------|
| $k_1, k_2, k_3$ | 방사 왜곡 | 값이 클수록 왜곡 심함 |
| $p_1, p_2$ | 접선 왜곡 | 센서 기울기 오차 |

## 8. 카메라 캘리브레이션이란?

### 8.1 정의

카메라의 **내부 파라미터 K** (fx, fy, cx, cy)와 **왜곡 계수** (k1, k2, p1, p2, k3)를 수학적으로 측정하는 과정

### 8.2 캘리브레이션 전후 차이

| 항목 | 캘리브레이션 전 | 캘리브레이션 후 |
|------|----------------|----------------|
| 직선 | 휘어져 보임 | 올바른 직선으로 보정 |
| 거리 측정 | 부정확 | 정확한 3D 좌표 계산 가능 |
| 물체 인식 | 위치 오차 발생 | 정밀한 위치 파악 가능 |

### 8.3 내부 파라미터 vs 외부 파라미터

| 종류 | 내용 | 특징 |
|------|------|------|
| **내부 파라미터** (Intrinsic) | K 행렬, 왜곡 계수 | 카메라 고유값 — 한 번만 측정 |
| **외부 파라미터** (Extrinsic) | 회전 R, 이동 t | 카메라 설치 위치/자세 — 움직이면 바뀜 |

## 9. 체커보드로 캘리브레이션하는 이유

체커보드(흑백 격자 패턴)를 사용하는 이유:

1. **명암 대비 명확** → 교차점(코너)을 서브픽셀 단위까지 정밀 감지 가능
2. **규칙적인 격자** → 실제 3D 좌표를 쉽게 정의 가능 (Z=0 평면)
3. **자동 인식** → `cv2.findChessboardCorners()`로 자동 검출

### 주의: 체커보드 코너 개수 세는 법

```
8×6 칸 체커보드의 경우:

□■□■□■□■         ← 6줄
■□■□■□■□
□■□■□■□■
■□■□■□■□
□■□■□■□■
■□■□■□■□

내부 교차점: (8-1) × (6-1) = 7 × 5 = 35개
                ↑ 칸 수가 아닌 내부 교차점 수!
```

```python
# 9×6 내부 교차점 → 10×7 칸의 체커보드
CHECKERBOARD = (9, 6)   # (가로 교차점, 세로 교차점)
```

### 올바른 촬영 방법

| 항목 | 권장 사항 |
|------|-----------|
| 장수 | **최소 10~20장** (왜곡 심한 렌즈는 최대 40장) |
| 각도 | **±15°~45°** 범위로 다양한 축 방향 기울이기 |
| 거리 | **가깝게~멀리** 다양하게 촬영 |
| 위치 | 이미지 **전체 영역**(특히 가장자리) 골고루 커버 |
| 체커보드 | **평평하고 단단한 판**에 부착 |
| 줌 | 촬영 중 **줌 수준 일정** 유지 |

> **왜 다양한 각도가 필요한가?**  
> 초점 거리는 체커보드와의 거리 변화에서, 왜곡 계수는 가장자리 촬영에서 정확히 계산됩니다.

## 10. OpenCV 캘리브레이션 3단계

### 전체 흐름

```
체커보드 이미지 (10~20장)
         │
         ▼
① findChessboardCorners()   → 픽셀 단위 코너 검출
         │
         ▼
② cornerSubPix()            → 서브픽셀 단위 정밀화
         │
         ▼
③ calibrateCamera()         → K 행렬 + 왜곡 계수 계산
         │
         ▼
     결과: mtx, dist, RMS Error
```

### 단계별 코드

```python
import cv2
import numpy as np
import glob

CHECKERBOARD = (9, 6)      # 내부 교차점 수
SQUARE_SIZE  = 0.025       # 한 칸 실제 크기 (m) — 2.5cm

# 3D 기준점 생성 (Z=0 평면)
objp = np.zeros((CHECKERBOARD[0] * CHECKERBOARD[1], 3), np.float32)
objp[:, :2] = np.mgrid[0:CHECKERBOARD[0], 0:CHECKERBOARD[1]].T.reshape(-1, 2)
objp *= SQUARE_SIZE

objpoints = []  # 3D 실세계 좌표 목록
imgpoints = []  # 2D 이미지 픽셀 좌표 목록

# cornerSubPix 종료 조건: 정확도 0.001 또는 최대 30회 반복
criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001)

for fname in glob.glob('calib_images/*.jpg'):
    img  = cv2.imread(fname)
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

    # ① 코너 탐색
    ret, corners = cv2.findChessboardCorners(gray, CHECKERBOARD, None)

    if ret:
        # ② 서브픽셀 정밀화 (윈도우 크기 11×11)
        corners_refined = cv2.cornerSubPix(
            gray, corners, (11, 11), (-1, -1), criteria
        )
        objpoints.append(objp)
        imgpoints.append(corners_refined)

# ③ 캘리브레이션 계산
ret, mtx, dist, rvecs, tvecs = cv2.calibrateCamera(
    objpoints, imgpoints, gray.shape[::-1], None, None
)
```

### calibrateCamera() 반환값 설명

| 반환값 | 형태 | 설명 |
|--------|------|------|
| `ret` | float | 재투영 오차 RMS (작을수록 좋음) |
| `mtx` | 3×3 | 카메라 행렬 K (fx, fy, cx, cy) |
| `dist` | 1×5 | 왜곡 계수 [k1, k2, p1, p2, k3] |
| `rvecs` | N×3 | 각 이미지의 회전 벡터 (외부 파라미터) |
| `tvecs` | N×3 | 각 이미지의 이동 벡터 (외부 파라미터) |

---

## 11. 재투영 오차 (Reprojection Error)

### 개념

계산된 카메라 모델로 3D 점을 다시 2D에 투영했을 때,  
실제 감지된 코너 위치와의 **픽셀 단위 거리 차이**

```
실제 감지된 코너 ●
                  ↑ 이 거리가 재투영 오차
재투영된 위치   ○
```

### 평가 기준

| RMS Error (픽셀) | 평가 |
|-----------------|------|
| < 0.5 px | 산업용 정밀 비전에서 성공적 |
| < 1.0 px | 대부분의 응용 분야에서 수용 가능 |
| ≥ 1.0 px | 촬영 데이터 재검토 후 재캘리브레이션 필요 |

### 재투영 오차 계산 코드

```python
errors = []
for i, (op, ip) in enumerate(zip(objpoints, imgpoints)):
    # 3D 점 → 2D 투영
    proj, _ = cv2.projectPoints(op, rvecs[i], tvecs[i], mtx, dist)
    # 실제 감지 좌표와의 거리 오차
    err = cv2.norm(ip, proj, cv2.NORM_L2) / len(proj)
    errors.append(err)

mean_error = np.mean(errors)
print(f"평균 재투영 오차: {mean_error:.4f} px")
```

## 12. 왜곡 보정 (Undistort)

### 캘리브레이션 → 왜곡 보정 흐름

```
실제 렌즈 카메라 (왜곡 있음)
        ↓
캘리브레이션으로 왜곡 계수 (k1, k2, p1, p2, k3) 측정
        ↓
undistort()로 왜곡 보정
        ↓
핀홀 카메라처럼 동작하는 이미지
```

### undistort() 사용법

```python
# mtx, dist: calibrateCamera()로 얻은 결과
dst = cv2.undistort(img, mtx, dist, None, mtx)

# 결과 시각화
import matplotlib.pyplot as plt
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 5))
ax1.imshow(cv2.cvtColor(img, cv2.COLOR_BGR2RGB))
ax1.set_title("원본 (왜곡 있음)")
ax2.imshow(cv2.cvtColor(dst, cv2.COLOR_BGR2RGB))
ax2.set_title("보정 후 (왜곡 제거)")
plt.show()
```

## 13. 캘리브레이션 결과 저장/불러오기

캘리브레이션은 **한 번 측정하면 재사용**합니다. 파일로 저장해 두세요.

### YAML 형식으로 저장 (ROS 호환)

```python
import yaml

def save_calibration(mtx, dist, filepath="camera_info.yaml"):
    data = {
        'camera_matrix': mtx.tolist(),
        'dist_coeff': dist.tolist()
    }
    with open(filepath, 'w') as f:
        yaml.dump(data, f)
    print(f"저장 완료: {filepath}")

save_calibration(mtx, dist)
```

### YAML 파일 불러오기

```python
def load_calibration(filepath="camera_info.yaml"):
    with open(filepath, 'r') as f:
        data = yaml.safe_load(f)
    mtx  = np.array(data['camera_matrix'], np.float64)
    dist = np.array(data['dist_coeff'], np.float64)
    return mtx, dist

mtx, dist = load_calibration()
```

## 15. 자주 하는 실수와 해결법

### 문제 1: 코너가 검출되지 않는다

```
원인: 패턴 크기 설정 오류, 이미지 흐림, 조명 문제
해결:
- CHECKERBOARD 값이 실제 체커보드와 일치하는지 확인
  (칸 수 - 1 = 교차점 수)
- 이미지가 선명한지 확인 (흔들림, 초점 맞지 않음 주의)
- 조명이 균일한지 확인 (역광, 반사 피하기)
```

### 문제 2: 재투영 오차가 1픽셀 이상이다

```
원인: 촬영 각도 편향, 이미지 수 부족, 체커보드 휘어짐
해결:
- 다양한 각도의 이미지 추가 촬영
- 오차가 큰 이미지 제거 후 재캘리브레이션
- 체커보드가 평평한지 확인
```

### 문제 3: 왜곡 보정 후 이미지가 이상하다

```
원인: 왜곡 계수 이상값, 촬영 이미지 부족
해결:
- dist 계수 값 확인 (k1이 ±1.0 초과하면 의심)
- 이미지 수 늘려서 재캘리브레이션
- 가장자리 영역이 포함된 이미지 추가
```

### 문제 4: 픽셀 → 3D 좌표 변환이 틀리다

```
원인: 깊이(Z) 정보 없음, 좌표계 축 혼용
해결:
- Z 없이는 정확한 역투영 불가 — 깊이 카메라 또는 물체 크기 기반 추정 사용
- 로봇 좌표계(X=앞)와 카메라 좌표계(Z=앞)가 다름을 주의
```

## 핵심 공식 요약

| 공식 | 설명 |
|------|------|
| $f_x = f / d_x$ | 픽셀 단위 초점 거리 |
| $u = f_x \cdot X/Z + c_x$ | 3D → 2D 투영 |
| $X = (u - c_x) \cdot Z / f_x$ | 2D → 3D 역투영 (Z 필요) |
| $Z = X_{real} \cdot f / (x_{px} \cdot d_x)$ | 닮음 삼각형 깊이 추정 |
| $K = [[f_x, 0, c_x], [0, f_y, c_y], [0, 0, 1]]$ | 카메라 행렬 |
