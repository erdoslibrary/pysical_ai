## 1. 컨볼루션(Convolution)과 패딩(Padding)

### 핵심 질문: "커널을 이미지에 어떻게 적용하는가?"

```python
def apply_kernel(img, kernel):
    h, w = img.shape
    k = kernel.shape[0]
    pad = k // 2
    padded = np.pad(img, pad, mode='reflect')  # ← 핵심!
    ...
```

### 패딩이 필요한 이유

3×3 커널을 가장자리 픽셀에 적용하면 **이미지 바깥 영역**이 필요합니다.

```
원본 이미지 (5×5):        커널을 (0,0)에 적용하려면?
┌─────────────┐           ┌───┬─────────────┐
│ A B C D E  │           │ ? │ ? ? ?       │  ← 바깥이 없다!
│ F G H I J  │     →     │ ? │ A B C       │
│ ...        │           │ ? │ F G H       │
```

**패딩 방식 3가지 비교:**

| 방식 | 설명 | 사용 상황 |
|------|------|----------|
| `zero` | 가장자리를 0으로 채움 | 검은 테두리 생길 수 있음 |
| `reflect` | 거울처럼 반사 (`DCBA│ABCDE`) | 블러·소벨에 자연스러움 ✓ |
| `edge` | 가장자리 값 그대로 복사 | 미디안 블러에 사용 |

> `reflect` 모드: OpenCV `cv2.blur`, `cv2.Sobel` 의 기본값과 동일하게 맞추기 위해 선택

### 컨볼루션 계산 원리

```
커널:          이미지 3×3 영역:     결과 픽셀:
[-1  0  1]     [10  10  200]
[-2  0  2]  ×  [10  10  200]  →  합산(요소별 곱) = 출력값
[-1  0  1]     [10  10  200]

즉: (-1×10) + (0×10) + (1×200)
  + (-2×10) + (0×10) + (2×200)
  + (-1×10) + (0×10) + (1×200) = 760
```

## 2. 블러 필터 4종 비교

### 2-1. 평균 블러 (Average Blur)

**원리:** 이웃 픽셀의 단순 평균

```python
kernel = np.ones((k, k)) / (k * k)   # 5×5라면 각 칸에 1/25
```

```
커널 (3×3):     모든 이웃이 동등한 가중치
[1/9  1/9  1/9]
[1/9  1/9  1/9]
[1/9  1/9  1/9]
```

- 장점: 가장 빠르고 단순
- 단점: 엣지도 같이 흐려짐 (중심과 먼 픽셀도 동일 가중치)

---

### 2-2. 가우시안 블러 (Gaussian Blur)

**원리:** 중심에서 멀수록 가중치가 줄어드는 정규분포 커널

```python
G(x, y) = (1 / 2πσ²) × exp(-(x² + y²) / (2σ²))
```

```
커널 (5×5, σ=1.0 예시):
[1  4  7  4  1]
[4 16 26 16  4]   ÷ 273  (합이 1이 되도록 정규화)
[7 26 41 26  7]
[4 16 26 16  4]
[1  4  7  4  1]
      ↑
  중심일수록 가중치가 크다
```

**σ(시그마)의 역할:**

```
σ 작음 → 뾰족한 분포 → 중심만 강조 → 약한 블러
σ 큼   → 넓은 분포   → 먼 픽셀도 반영 → 강한 블러
```

**sigma=0 일 때 OpenCV 공식:**
```python
sigma = 0.3 * (k // 2 - 1) + 0.8
# k=5 → sigma = 0.3 × 1 + 0.8 = 1.1
```

- 장점: 자연스러운 블러, 구조 보존
- **엣지 검출 전처리의 표준**

---

### 2-3. 미디안 블러 (Median Blur)

**원리:** 이웃 픽셀을 정렬한 뒤 **중간값**으로 대체

```python
region = [0, 120, 122, 118, 255, 119, 121, 117, 255]
→ 정렬: [0, 117, 118, 119, 120, 121, 122, 255, 255]
→ 중간값: 120  ← 극단값(0, 255)이 결과에 영향 없음!
```

```
소금-후추 노이즈 예시:
  원본:    [ 10  255   12  0   11 ]   ← 255, 0 이 노이즈
  미디안:  [ 10   11   12  11  10 ]   ← 자연스럽게 제거됨!
  평균:    [ 57   57   ...          ]  ← 평균은 노이즈가 결과를 오염
```

- 장점: **점(impulse) 노이즈 제거에 탁월**
- 단점: 계산량이 많음 (매번 정렬 필요)

---

### 2-4. 바이래터럴 필터 (Bilateral Filter) ⭐ 고급

**핵심 아이디어:** 공간 거리 + 밝기 차이, **두 조건을 동시에** 고려

```python
w(y, x) = exp(-거리²  / (2·σ_space²))   # 공간 가중치
         × exp(-밝기차² / (2·σ_color²))  # 색상 가중치
```

**가우시안 블러와의 결정적 차이:**

```
가우시안:
  중심 픽셀(밝기=50)과 이웃(밝기=200)  →  거리 가중치만 → 같이 평균됨 → 엣지 흐려짐

바이래터럴:
  중심 픽셀(밝기=50)과 이웃(밝기=200)  →  밝기 차이(150)가 크므로 → 가중치 거의 0
  → 이 이웃을 평균에서 거의 제외           → 엣지 보존!
```

```
σ_space 큼  → 공간적으로 넓은 범위 참고 (더 많이 블러)
σ_color 큼  → 밝기 차이가 커도 평균에 포함 (엣지도 흐려짐)
σ_color 작음 → 밝기 차이가 조금만 나도 제외 (엣지 더 잘 보존)
```

| 파라미터 | 역할 |
|---------|------|
| `d=9` | 이웃 범위 (원형 마스크 반지름) |
| `sigmaColor=75` | 밝기 차이 허용 범위 |
| `sigmaSpace=75` | 공간 거리 가중치 σ |

- 장점: **엣지를 살리면서 노이즈만 제거** (HDR·인물사진 보정에 사용)
- 단점: 계산량이 가장 많음 (실시간 로봇에서는 주의)

---

### 필터 선택 요약

```
노이즈 종류가 뭔가?
    │
    ├── 소금-후추(점 노이즈)  →  미디안 블러
    │
    ├── 가우시안(전반적 잡음)  →  가우시안 블러 (엣지 검출 전처리 표준)
    │
    ├── 엣지를 꼭 보존해야 함  →  바이래터럴 필터
    │
    └── 속도 최우선            →  평균 블러
```

## 3. 라플라시안 엣지 검출 (Laplacian)

### 소벨(1차 미분) vs 라플라시안(2차 미분)

```
밝기 프로파일:    ─────╱╲─────
1차 미분(Sobel):  ──/─╲──   ← 기울기 변화를 보여줌
2차 미분(Lapl.):  ─/╲ ╱╲╲─  ← 기울기의 기울기, Zero-crossing 지점이 엣지
```

**라플라시안 커널:**
```
[ 0   1   0 ]
[ 1  -4   1 ]   ← 중심은 음수, 이웃의 합 - 4×중심 = 2차 미분 근사
[ 0   1   0 ]
```

**세 알고리즘 비교:**

| | Sobel | Laplacian | Canny |
|--|-------|-----------|-------|
| 미분 차수 | 1차 | 2차 | 1차 + 후처리 |
| 방향성 | O (Gx, Gy 별도) | X (모든 방향) | O (NMS에서 활용) |
| 노이즈 강인성 | 낮음 | 매우 낮음 | **높음** |
| 엣지 두께 | 두꺼움 | 얇음 | **1픽셀** |
| 실무 사용 | 보통 | 드묾 | **표준** |

> 라플라시안은 2차 미분이므로 작은 노이즈도 크게 증폭됩니다.  
> **반드시 가우시안 블러 전처리 후** 사용해야 합니다.

## 4. 엣지 → 객체화 파이프라인 ⭐ 로봇 핵심

엣지를 검출했다고 끝이 아닙니다.  
실제 로봇은 **"어디에 뭐가 있다"** 는 정보가 필요합니다.

### 전체 파이프라인

```
컬러 이미지
    ↓ cv2.cvtColor(BGR2GRAY)
그레이스케일
    ↓ cv2.GaussianBlur(gray, (5,5), 0)
블러 이미지
    ↓ cv2.Canny(blurred, 50, 150)
엣지 이미지 (흰선)
    ↓ cv2.findContours(edges, RETR_EXTERNAL, CHAIN_APPROX_SIMPLE)
윤곽선 목록 (contours)
    ↓ cv2.contourArea(cnt) > 900  →  작은 노이즈 제거
    ↓ cv2.boundingRect(cnt)
바운딩 박스 (x, y, w, h)
    ↓ cv2.rectangle(result, ...)
최종 결과 (객체 위치 표시)
```

### findContours 파라미터 이해

```python
contours, _ = cv2.findContours(
    edges,
    cv2.RETR_EXTERNAL,       # 가장 바깥 윤곽선만 (내부 구멍 무시)
    cv2.CHAIN_APPROX_SIMPLE  # 직선 방향 중간 점은 생략 (메모리 절약)
)
```

**RETR 모드:**
```
원본 이미지: 바깥 사각형 안에 안쪽 사각형 있을 때

RETR_EXTERNAL:  ▭  ← 바깥 사각형만
RETR_LIST:      ▭ □  ← 모든 윤곽선
RETR_TREE:      계층 구조로 모두
```

### 면적 필터링

```python
for cnt in contours:
    if cv2.contourArea(cnt) > 900:  # 30×30 픽셀 이상만
        x, y, w, h = cv2.boundingRect(cnt)
        cv2.rectangle(result, (x, y), (x+w, y+h), (0, 255, 0), 2)
```

면적 임계값(900)을 조절해 노이즈 제거 강도를 바꿀 수 있습니다.

## 5. Canny 임계값 튜닝 전략

### 권장 비율 2:1 또는 3:1

```python
# 좋은 예 (3:1 비율)
cv2.Canny(blurred, 30, 90)    # 민감 — 약한 엣지까지 검출
cv2.Canny(blurred, 50, 150)   # 균형 — 표준
cv2.Canny(blurred, 100, 300)  # 강인 — 뚜렷한 엣지만
```

```
low 낮음 → 더 많은 엣지 (노이즈 포함 가능)
low 높음 → 적은 엣지 (중요한 엣지도 끊길 수 있음)

high 낮음 → strong 엣지 씨앗이 너무 많음
high 높음 → strong 엣지 씨앗이 너무 적음 → 이력 현상이 퍼지지 않음
```

**실무 튜닝 방법:**
1. 먼저 `high = 150`, `low = 50` 으로 시작
2. 엣지가 너무 많으면 두 값 모두 올림
3. 엣지가 끊기면 두 값 모두 내림
4. 비율(2:1 또는 3:1)은 유지

## 6. 실제 로봇 환경 주의사항

### 흔한 실수 3가지

```
❌ 실수 1: 전처리 없이 Canny 바로 적용
   gray → Canny()  →  노이즈가 모두 엣지로 검출됨

✅ 올바른 순서:
   gray → GaussianBlur → Canny

---

❌ 실수 2: 컬러 이미지에 Canny 적용
   image_bgr → Canny()  →  오류 또는 이상한 결과

✅ 올바른 순서:
   image_bgr → cvtColor(BGR2GRAY) → GaussianBlur → Canny

---

❌ 실수 3: 블러 후에 또 블러
   blur → Canny → GaussianBlur  →  의미 없음, 엣지 더 흐려짐

✅ 순서:
   전처리 → 검출 → 시각화
```

### 환경별 대처법

| 환경 문제 | 대처 방법 |
|---------|---------|
| 조명 변화 | HSV 색공간 V채널 활용, 적응형 임계값 |
| 그림자 | 색상 채널 분리 처리 |
| 실시간 처리 | 바이래터럴 대신 가우시안 블러 사용 |
| 동적 임계값 | Otsu 자동 임계값 (`cv2.threshold(..., cv2.THRESH_OTSU)`) |

## 퀴즈 정답 해설

**Q1.** `cv2.GaussianBlur(img, (5, 5), 0)` 에서 `(5, 5)` 는?
> 커널 크기 = 5×5. 홀수여야 중심 픽셀이 명확함. 클수록 더 많이 흐려짐.

**Q2.** 수직 방향 경계선(수평 밝기 변화) 강조?
> A: `dx=1, dy=0` → **Kx 커널** → 수평 방향 그래디언트 → **수직 엣지** 검출

**Q3.** 순서 `Canny → GaussianBlur` 가 올바른가?
> **올바르지 않음.** 반드시 `GaussianBlur → Canny` 순서여야 합니다.  
> Canny는 내부적으로 블러를 포함하지만, 외부 블러로 노이즈를 먼저 제거해야 더 정확합니다.
