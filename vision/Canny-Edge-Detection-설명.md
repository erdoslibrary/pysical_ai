## 0. 왜 엣지(Edge)를 검출하는가?

이미지에서 **엣지**란 밝기(intensity)가 급격히 변하는 경계선입니다.  
물체의 윤곽, 텍스처 경계, 그림자 등이 모두 엣지로 나타납니다.

```
어두운 영역 → → → 밝은 영역
   낮은 픽셀값   급격한 변화   높은 픽셀값
                    ↑
                  엣지!
```

엣지를 찾으면 → 물체 인식, 형태 분석, 깊이 추정 등이 쉬워집니다.

Canny 엣지 검출은 이 작업을 **4단계**로 나누어 매우 정밀하게 수행합니다.

---

## 1단계: 소벨 그래디언트 (Sobel Gradient)

### 핵심 질문: "밝기가 얼마나 빠르게 변하는가?"

**그래디언트(Gradient)** 는 수학적으로 함수의 변화율입니다.  
이미지에서는 픽셀 밝기의 변화량 = **미분(derivative)** 을 의미합니다.

### 소벨 커널(Kernel)이란?

소벨 커널은 이미지를 **합성곱(Convolution)** 하여 미분 근사값을 계산합니다.

```python
Kx = [[-1, 0, 1],   # 수평 방향(좌→우) 밝기 변화 감지
      [-2, 0, 2],
      [-1, 0, 1]]

Ky = [[-1, -2, -1], # 수직 방향(위→아래) 밝기 변화 감지
      [ 0,  0,  0],
      [ 1,  2,  1]]
```

#### Kx 직관적 이해

```
픽셀 3×3 영역을 본다고 가정:

[ 10  10  200 ]       왼쪽은 어둡고 오른쪽은 밝다
[ 10  10  200 ]  →  수평 방향으로 큰 변화 → Gx 값이 크다!
[ 10  10  200 ]

Kx 적용 결과 = (-1×10 + 0×10 + 1×200)   (첫 번째 행)
             + (-2×10 + 0×10 + 2×200)   (두 번째 행, 중심 가중치 2배)
             + (-1×10 + 0×10 + 1×200)   (세 번째 행)
             = 190 + 380 + 190 = 760  → 큰 수평 그래디언트!
```

중심 행에 가중치 **2**를 주는 이유: 중심 픽셀이 더 중요하기 때문입니다 (노이즈 감소 효과).

#### Ky 직관적 이해

Ky는 Kx를 90도 회전한 형태로, **수직 방향** 밝기 변화를 감지합니다.

### 그래디언트 크기와 방향 계산

```python
Gx = my_sobel_conv(blurred, Kx)   # 수평 방향 변화량
Gy = my_sobel_conv(blurred, Ky)   # 수직 방향 변화량

G     = np.sqrt(Gx**2 + Gy**2)   # 전체 변화량 (크기)
theta = np.arctan2(Gy, Gx) * 180 / np.pi  # 변화 방향 (각도)
```

#### 벡터로 생각하기

각 픽셀에서 `(Gx, Gy)`는 **2D 벡터**입니다.

```
      Gy(↑)
       |
       |  /← 그래디언트 벡터 (Gx, Gy)
       | /
       |/_____ Gx(→)

크기 G = √(Gx² + Gy²)  = 벡터의 길이 (얼마나 급하게 변하는가)
방향 θ = arctan2(Gy, Gx) = 벡터의 각도 (어느 방향으로 변하는가)
```

> **직관:** 엣지에 수직인 방향 = 밝기가 가장 빠르게 변하는 방향 = 그래디언트 방향

### 각도 정규화 (0° ~ 180°)

```python
theta = (theta + 180) % 180   # -180~180 → 0~180 으로 변환
```

엣지는 방향이 있어도 **양방향이 동일**합니다.  
(왼쪽→오른쪽 엣지 = 오른쪽→왼쪽 엣지는 같은 엣지)  
따라서 180도 범위로 충분합니다.

## 2단계: 비최대 억제 (Non-Maximum Suppression, NMS)

### 핵심 질문: "이 픽셀이 진짜 엣지의 최고점인가?"

소벨 그래디언트만 쓰면 엣지가 **여러 픽셀 두께**로 번집니다.  
NMS는 엣지를 **1픽셀 두께**로 얇게 만들어줍니다.

### 원리

그래디언트 **방향을 따라** 이웃 픽셀과 비교합니다.  
주변보다 크지 않으면? → 엣지가 아니므로 0으로 제거합니다.

```
그래디언트 방향이 수평(→)일 때:

  [ ... | 80 | 150 | 90 | ... ]
              ↑
         이 픽셀이 이웃(80, 90)보다 크므로 → 살린다!
         만약 100이었다면(80<100<150) → 진짜 피크가 아니므로 제거
```

### 4방향 양자화

방향을 계속 **4가지 축**으로 단순화합니다.

```python
if (ang < 22.5) or (ang >= 157.5):    # ─  수평 방향
    n1, n2 = G[y, x-1], G[y, x+1]

elif ang < 67.5:                       # ╲  45° 대각선
    n1, n2 = G[y-1, x-1], G[y+1, x+1]

elif ang < 112.5:                      # │  수직 방향
    n1, n2 = G[y-1, x], G[y+1, x]

else:                                  # ╱  135° 대각선
    n1, n2 = G[y-1, x+1], G[y+1, x-1]
```

```
       135° ╲  │ 90°  ╱ 45°
              \ | /
        ───────●───────  0°/180°
              / | \
```

### 동률 처리 (>=, > 혼합)

```python
if g >= n1 and g > n2:   # 한쪽은 >=, 한쪽은 >
    nms[y, x] = g
```

두 픽셀이 **완전히 같은 값**일 때 둘 다 살리면 2픽셀 두께가 됩니다.  
한쪽만 `>`로 검사해서 이를 방지합니다.

## 3단계: 이중 임계값 (Hysteresis Thresholding)

### 핵심 질문: "이 엣지가 진짜인가, 노이즈인가?"

임계값 하나만 쓰면?
- 너무 높으면 → 진짜 엣지도 끊겨 사라집니다.
- 너무 낮으면 → 노이즈도 엣지로 포함됩니다.

해결책: 임계값을 **두 개** 사용합니다.

```python
low, high = 50, 150

strong = nms >= high          # 확실한 엣지
weak   = (nms >= low) & (nms < high)  # 애매한 엣지
```

```
  G값:  0 ────────── 50 ────────── 150 ──────→
        |  노이즈    | 약한 엣지  | 강한 엣지
        |  (버림)    | (조건부)   | (확실히 살림)
```

### 이력 현상 (Hysteresis) — 핵심 아이디어

**약한 엣지**는 **강한 엣지와 연결된 경우에만** 살립니다!

```python
changed = True
while changed:
    changed = False
    for y in range(1, H - 1):
        for x in range(1, W - 1):
            if weak[y, x] and edges_np[y, x] == 0:
                # 8방향 이웃 중 강한 엣지가 하나라도 있으면
                if (edges_np[y-1:y+2, x-1:x+2] == 255).any():
                    edges_np[y, x] = 255   # 이 약한 엣지도 살린다
                    changed = True
```

#### 시각적 설명

```
before:
  S S S . . . W W W . . .
  (S=강함, W=약함, .=없음)

after:
  S S S W W W . . . . . .
  ↑___연결됨___↑ 살림!  ↑ 고립됨 → 버림
```

`while changed` 루프: 한 번에 하나씩 전파되므로,  
긴 약한 엣지 체인도 **반복을 통해 끝까지 추적**합니다.

## 전체 흐름 요약

```
원본 이미지
    ↓
[가우시안 블러] — 노이즈 제거
    ↓
[소벨 그래디언트] — 밝기 변화량(크기 G)과 방향(θ) 계산
    ↓
[비최대 억제] — 엣지를 1픽셀 두께로 가늘게
    ↓
[이중 임계값 + 이력 현상] — 진짜 엣지만 선별
    ↓
최종 엣지 이미지
```

## 핵심 정리

| 단계 | 목적 | 핵심 연산 |
|------|------|----------|
| 소벨 그래디언트 | 밝기 변화 측정 | `G = √(Gx²+Gy²)`, `θ = arctan2(Gy, Gx)` |
| 비최대 억제 | 엣지 얇게 만들기 | 그래디언트 방향 이웃과 비교 |
| 이중 임계값 | 노이즈와 엣지 구분 | `strong(≥150)`, `weak(50~150)` |
| 이력 현상 | 약한 엣지 연결 추적 | 8-연결 이웃 탐색 반복 |
